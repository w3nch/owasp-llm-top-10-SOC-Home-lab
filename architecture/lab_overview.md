# Lab Overview

This lab is designed to study security risks in **Large Language Model (LLM) applications** and build basic monitoring and detection capabilities for them.  
The environment focuses on running **local LLM models with Ollama**, simulating attacks from the **OWASP Top 10 for LLM Applications**, and observing activity using **Wazuh**.

The goal is to understand how attackers may abuse LLM systems and how defenders can monitor and detect suspicious model interactions.


## Core Technologies

The lab environment is built around three main components.

### Ollama

Ollama is used to run **LLM models locally** on the system.  
Running models locally provides full control over prompts, responses, and system behavior, which makes it suitable for security research and experimentation.

Models are accessed through the local Ollama API and can be interacted with through command line tools or web interfaces.


### LLM Models

Several lightweight models are used so the lab can run on systems with limited hardware resources.

Example models used in this lab:

tinyllama  
phi  
deepseek-coder:1.3b

These models allow testing of different types of prompts, reasoning tasks, and code generation behavior.


### Wazuh

Wazuh is used as the **security monitoring and detection platform** for the lab.

It can be used to:

• monitor system activity  
• analyze logs generated by the LLM environment  
• detect suspicious prompts or interactions  
• generate alerts for potential abuse

Wazuh helps simulate how a **SOC environment might monitor AI systems** for malicious behavior.


## OWASP Top 10 for LLM Applications

This lab focuses on experimenting with attack techniques described in the **OWASP Top 10 for LLM Applications**.

Examples of attacks explored in the lab include:

Prompt Injection  
System Prompt Leakage  
Jailbreak Attacks  
RAG Data Exfiltration  
Agent Tool Abuse

These attacks attempt to manipulate the model into ignoring safeguards, revealing hidden instructions, or generating unintended outputs.


## Lab Workflow

A typical workflow in this lab looks like the following:

1. Start the Ollama service and load a model  
2. Interact with the model using prompts  
3. Attempt attack scenarios based on OWASP LLM vulnerabilities  
4. Monitor system activity and logs with Wazuh  
5. analyze suspicious prompts and model responses


## Purpose of the Lab

This lab provides a simple environment to experiment with:

• LLM security risks  
• attack techniques against AI systems  
• monitoring and detection strategies  
• security visibility into model interactions
